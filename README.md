# TriageFlow：基于 LLM 的门诊预检分诊与智能排队系统

## 1. 项目概述

### 1.1 项目名称
TriageFlow

### 1.2 一句话介绍
面向门诊场景，利用 LLM 完成患者主诉理解和辅助分诊，并通过规则引擎保障高危场景安全兜底，实现智能预检、优先级排队和实时叫号。

### 1.3 项目定位
这是一个 **LLM 辅助预检分诊系统**，不是自动诊断系统，也不替代医生决策。

系统目标：
- 患者输入主诉和基础信息
- LLM 提取结构化症状、风险信号、候选科室和建议优先级
- 规则引擎做高危兜底和最终优先级裁决
- 进入排队系统
- 护士/医生工作台查看结果并处理

### 1.4 项目边界
明确限制：
- 不输出疾病诊断结果
- 不输出治疗方案
- 不输出药物建议
- 仅作为辅助预检和排队分流工具
- 高风险场景必须人工复核

## 3. 核心设计原则

### 3.1 分层设计
- LLM 负责自然语言理解和辅助建议
- 规则引擎负责医疗安全边界和最终裁决
- 排队服务负责候诊顺序和叫号逻辑

### 3.2 医疗安全原则
- 高危症状优先走规则，不完全相信 LLM
- 保留完整审计日志
- 支持人工复核
- 所有分诊结果可追踪

### 3.3 Demo 优先原则
项目目标是做出一个：
- 可以跑
- 可以演示
- 可以讲清楚
- 有 AI 亮点
- 有业务闭环

## 4. 用户角色

### 4.1 患者端
功能：
- 填写基础信息
- 输入主诉
- 回答补充问题
- 查看分诊结果
- 查看排队进度

### 4.2 护士/分诊台
功能：
- 查看新患者
- 查看 AI 分诊建议
- 查看规则命中情况
- 人工修正分诊结果
- 管理叫号

### 4.3 医生端
功能：
- 查看待诊患者
- 查看分诊摘要
- 开始接诊
- 完成接诊
